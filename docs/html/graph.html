Graphs¶

The Graph class provides a graph object and it’s methods.

class db.Graph(backend=None)¶

Create a graph object

Parameters:backend (optional) – Backend object. Defaults to an im-memory

DictionaryBackend.

Note

The graph object is iterable! Useing it as an iterator provides all the elements (nodes and edges) of the graph!

nodes¶

iterable of all the nodes in the graph

edges¶

iterable of all the edges of the graph

__getitem__(element)¶
Returns:A dictionary containing all the attributes of element
Parameters:element – An element(node or edge) object that belongs to the graph
__iter__()¶
Returns:An iterator over all graph elements (nodes and edges).

Note

To get just nodes or edges, use the nodes or edges properties.

__setitem__(element, items)¶

Set attributes for a node or edge.

Parameters:
  • element – An element (node or edge) object that belongs to the Graph
  • items – A dictionary containing the attributes you want to set
Returns:

A dictionary containg all of the elements attributes after updating

Note

The attributes in items are added with the update() method, so duplicate keys will be overwritten, and new keys added, but existing keys which aren’t duplicated in item won’t be overwritten.

the edge object

add_nodes(num, attributes=None, *args, **kwargs)¶

Adds number of nodes specified in num

Parameters:
  • num – is an int representing the number of nodes to add
  • attributes (optional) – is a dictionary of attributes to assign to each node

Note

If you supply attributes, each node will have an identical copy of attributes applied to it. You cannot use this method to specify different attributes for each node.

Returns:a list containing the new node objects
